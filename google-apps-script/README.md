# 培兒蒂預約表單 — Google Sheet 串接設定步驟

`contact.html` 的預約表單送出後，會透過 Google Apps Script 把資料寫入 Google Sheet。以下是完整設定流程。

## 1. 建立 Google Sheet

1. 開啟 [Google Sheets](https://sheets.google.com)，新增一份空白試算表
2. 幫試算表命名，例如「培兒蒂預約名單」
3. 不需要手動建立欄位標題，Apps Script 第一次寫入時會自動建立分頁「預約資料」並加上標題列：
   `送出時間 / 家長姓名 / 電子郵件 / 聯繫電話 / 孩子年齡 / 感興趣的服務 / 關心的議題`

## 2. 貼上並部署 Apps Script

1. 在該試算表上方選單點「擴充功能」>「Apps Script」，會開啟程式碼編輯器
2. 把預設的 `Code.gs` 內容全部刪除，改貼上本資料夾的 `Code.gs` 內容
3. 點右上角「儲存」（磁片圖示）
4. 點「部署」>「新增部署作業」
   - 類型選擇「網頁應用程式」
   - 說明可填「培兒蒂預約表單」
   - 執行身分：**我**
   - 誰可以存取：**任何人**（一定要選這個，否則前端無法呼叫）
5. 點「部署」，第一次會要求授權，依畫面指示登入並允許權限
6. 部署完成後會顯示一組「網頁應用程式網址」，格式類似：
   `https://script.google.com/macros/s/AKfycb.../exec`
   複製這組網址

## 3. 把網址填入 HTML

打開 `contact.html`，找到接近檔案最下方的這一行：

```js
const GOOGLE_SHEET_WEB_APP_URL = 'https://script.google.com/macros/s/請貼上你的部署ID/exec';
```

把整段網址換成步驟 2 複製到的部署網址。

> 之後如果修改了 `Code.gs` 的內容，需要重新「新增部署作業」或在既有部署上「管理部署作業」>「編輯」>更新版本，否則正式環境不會套用新程式碼。

## 4. 測試是否成功

1. 直接在瀏覽器打開部署網址（`.../exec`），若看到
   `{"success":true,"message":"培兒蒂預約表單服務運作中..."}`
   代表服務本身部署成功
2. 用 `python3 -m http.server 8080` 在專案根目錄啟動本機伺服器，瀏覽 `http://localhost:8080/contact.html`
3. 填寫表單（記得留意必填欄位：家長姓名、電子郵件、聯繫電話），送出後：
   - 按鈕會顯示「預約發送中...」並暫時無法再次點擊
   - 成功後畫面顯示「已收到您的預約！」，表單自動清空
   - 回到 Google Sheet 的「預約資料」分頁，應該會看到新增的一列資料
4. 若顯示紅色的「送出失敗」訊息，常見原因：
   - 網址還沒換成實際部署網址
   - 部署時「誰可以存取」沒有選「任何人」
   - Apps Script 尚未完成授權流程

## 注意事項

- `Code.gs` 內建後端驗證（必填欄位、Email 格式），前端就算被繞過也不會寫入明顯不完整的資料
- 每次修改 `Code.gs` 後記得重新部署，否則使用者送出的資料不會套用新邏輯
- 若日後要新增欄位，同時要修改 `Code.gs` 的 `doPost`／`getOrCreateSheet` 與 `contact.html` 的 `collectFormData()`
